Build Tools, Package Managers and Modules

Build tools and package managers help developers manage dependencies, organize projects and automate development tasks. They are an important part of modern JavaScript and Node.js development.

Build Tools and Package Managers

  • With Build tools you can automate tasks such as bundling, compiling and optimizing code.
  • Use Package managers to install and manage project dependencies.
  • For example npm, yarn and pnpm.

package.json

The package.json file contains metadata about a project. It defines dependencies, scripts, project information and configurations. 


{
  "name": "my-project",
  "version": "1.0.0",
  "scripts": {
    "start": "node app.js"
  }
}

node_modules

The node_modules folder contains all installed dependencies. This folder can become very large and should not be committed to Git.

  • Dependencies are defined in package.json.
  • They can be reinstalled with npm install.
  • The folder is usually ignored via .gitignore.

Nodemon

Nodemon is a development tool that automatically restarts a Node.js server when code changes are detected. 


npm install nodemon --save-dev
  • Useful for development workflows.
  • But not intended for production environments.

Modules in JavaScript

Modules allows code to be split into reusable files. They help organize large applications and improve maintainability.

CommonJS vs ES Modules

  • CommonJS is the traditional Node.js module system.
  • ES Modules (ESM) are the modern JavaScript standard.

CommonJS example:


const math = require("./math.js");

ES Modules example:


import { add } from "./math.js";

Advantages of ES Modules are fx better static analysis, improved tooling support and compatibility with modern JavaScript.

Using ES Modules in the Browser

In the frontend, ES modules can be enabled by adding the type="module" attribute to a script tag.
