REST API and Express

REST (Representational State Transfer) is a set of conventions for designing web APIs. It defines how clients interact with servers and how resources are represented and accessed over HTTP.

REST API Conventions

  • APIs expose resources through URLs.
  • HTTP methods define the action performed on a resource.
  • Responses are usually sent as JSON.
  • APIs should be stateless, meaning each request contains all necessary information.

Richardson Maturity Model

The Richardson Maturity Model measures how closely an API follows REST principles. It has four levels:

  • Level 0 – Single endpoint with multiple operations.
  • Level 1 – Resources are separated into different URLs.
  • Level 2 – Proper use of HTTP methods like GET, POST, PUT, and DELETE.
  • Level 3 – Hypermedia links (HATEOAS) guide clients through the API.

Sending data with GET requests

GET requests retrieves data from the server. Parameters are usually sent in the URL as query parameters. 


GET /users?id=5

Express and creating a Server

Express is a Node.js framework used to build web servers and APIs. A basic server can be created by importing Express and defining routes. 


const express = require("express");
const app = express();

app.listen(3000, () => {
  console.log("Server running on port 3000");
});

Handling GET requests in Express

You can create routes that respond to GET requests and return data to the client. 


app.get("/users", (req, res) => {
  res.send("List of users");
});

Serving HTML with Express

Express can also serve HTML files to the browser. 


app.get("/", (req, res) => {
  res.sendFile(__dirname + "/index.html");
});

POST requests and parsing the body

POST requests send data to the server. In Express, middleware like express.json() is used to parse incoming request bodies. 


app.use(express.json());

app.post("/users", (req, res) => {
  const user = req.body;
  res.send("User received");
});

CRUD operations in a REST API

  • GET – Retrieve data
  • POST – Create new data
  • PUT / PATCH – Update existing data
  • DELETE – Remove data

Serving static files

Express can serve static files like CSS, images and JavaScript. This should be done carefully to avoid exposing sensitive files or information. 


app.use(express.static("public"));
