Client-Server Model and Deployment

The client-server model is a software architecture that separates the client (user interface) from the server (backend logic and data storage). The client sends requests to the server and the server processes those requests and returns responses.

Client-Server Model

  • Client – The part of the application the user interacts with (fx browser, app).
  • Server – Handles logic, processes requests and manages data.
  • Request – A message sent from the client to the server.
  • Response – The data returned from the server to the client.

This architecture allows many clients to communicate with a single server and helps separate frontend and backend responsibilities.

URL Terminology

A URL (Uniform Resource Locator) identifies the location of a resource on the web.


https://example.com:443/users?id=5
  • Protocol – https
  • Domain – example.com
  • Port – 443
  • Path – /users
  • Query parameters – ?id=5

Ways to send data from client to server

  • Query parameters – Sent in the URL with GET requests.
  • Request body – Sent with POST, PUT or PATCH requests.
  • Headers – Metadata such as authentication tokens.
  • Cookies – Stored in the browser and sent automatically.

Cross-Site Scripting (XSS)

Cross-Site Scripting (XSS) is a security vulnerability where malicious JavaScript is injected into a website and executed in another user's browsers.

  • Never trust user input.
  • Escape or sanitize input before rendering it.
  • Avoid inserting raw HTML from users.

Deployment

Deployment is the process of making an application available on the internet. This usually involves running the server on a hosting platform.

  • Upload code to a server or cloud service.
  • Install dependencies.
  • Run the application.

Client-Side vs Server-Side Redirection

Client-side redirection happens in the browser using JavaScript:


window.location.href = "/home";

Server-side redirection happens on the server before the response is sent:


res.redirect("/home");
